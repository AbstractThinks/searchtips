github仓库查找资料

一、查找用户或组织

更多高级搜索功能请参考github search help

搜索用户或组织

默认搜索会返回用户个人基本信息和国籍。但是，你也可以使用type限定符过滤个人账户和组织

tom in:email type:org

--查找email中含有tom的组织

mike in:name created:<2011-01-01 type:user

--查找在2011年以前创建的叫mike的个人账户

条件搜索

通过限定符设置可以限制搜索条件。在没有限定符的情况下，只能搜索用户名和email地址。 tom in:email

--查找email中包含tom的用户

kenya in:login

--查找用户名中包含kenya的用户

bolton in:fullname

--查找real name中包含bolton的用户

基于用户拥有的repositories数量的搜索

repos限定符可以基于用户含有的repositories数量进行查找用户，例如：

repos:>9000

--查找repository数量超过9000的用户

bert repos:10..30

--查找用户名或real name中包含bert关键字并且拥有的repositories的数量在10到30之间的用户

基于用户所在地的搜索

你可以通过用户个人信息中的所在地进行搜索。例如：

repos:1 location:iceland

--查找repository数量为1并且所在地为Iceland的用户

基于用户仓库所用编程语言的搜索

你可以通过language限定符搜索用户repositories中项目所使用的编程语言

language:javascript location:russia

--查找repositories中项目使用JavaScript编程语言，并且所在地为Russia的用户

jenny language:javascript in:fullname

--查找全名中包含jenny，并且repositories中项目使用JavaScript编程语言的用户

基于用户创建时间的搜索

created:<2011-01-01

--查找创建于2011-01-01的用户

created:>=2013-05-11

--查找创建于2013-05-11及以后的用户

created:2013-03-06 location:london

--查找创建于2013-03-06，所在地为london的用户

created:2010-01-01..2011-01-01 john in:login

--查找创建于2010-01-01--2011-01-01，用户名中包含john的用户

基于用户followers数量的搜索

你可以通过followers限定符进行搜索

followers:>=1000

--查找含有超过1000followers的用户

sparkle followers:1..10

--查找含有1到10followers的用户

二、查找repositories

范围搜索

jquery in:name

--查找名字中含有jquery的repositories

jquery in:name,description

--查找名字和说明含有jquery的repositories

jquery in:readme

--查找README文件含有jquery的repositories

基于repository大小的搜索

size:1000

--查找大小为1 MB的repositories

size:>=30000

--查找大小至少为30MB的repositories

size:<50

--查找大小低于50KB的repositories

size:50..120

--查找大小在50KB到120KB的repositories

基于repository大小的搜索

size:1000

--查找大小为1 MB的repositories

size:>=30000

--查找大小至少为30MB的repositories

size:<50

--查找大小低于50KB的repositories

size:50..120

--查找大小在50KB到120KB的repositories

基于repository被forks的数量的搜索

forks:5

--查找forks数量为5的repositories 

forks:>=205

--查找forks数量至少为205的repositories 

forks:<90

--查找forks数量小于90的repositories 

forks:10..20

--查找forks数量在10到20之间的repositories 

github fork:true

--查找包括forked在内，包含字符串github的repositories 

github fork:only

--查找forked的，包含字符串github的repositories 

github

--查找不包括forked在内，包含字符串github的repositories

基于repository被stars的数量的搜索

stars:10..20

--查找stars数量为10--20并且小于1000KB的repositories 

stars:>=500 fork:true language:php

--查找包括forked的repositories在内，stars数量至少为500的repositories

基于用户或组织的repository搜索

user:github

--查找github的repositories

user:defunkt forks:>100

--查找forks数量超过100的defunkt的repositories

基于编程语言的repository搜索

rails language:javascript

--查找含有rails并且编程语言为javascript的repositories

基于repository创建时间及更新时间的搜索

webos created:<2011-01-01

--查找含有关键字webos并且创建于2011年以前的repositories

css pushed:<2013-02-01

--查找含有关键字css并且更新于2013-02-01的repositories

case pushed:>=2013-03-06 fork:only

--只查找fork repositories中含有关键字case并且更新时间早于2013-03-06的repositories
